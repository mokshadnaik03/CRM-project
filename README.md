# WooCRM - Customer Relationship Management Dashboard

WooCRM is a modern, responsive Customer Relationship Management (CRM) dashboard built to manage customers, track orders, monitor communications, and leverage Artificial Intelligence for customer segmentation and analysis.

## đź”Ś APIs Used

### Google Gemini API
* **SDK**: `@google/genai`
* **Model**: `gemini-3-flash-preview`
* **Purpose**: The Gemini API is integrated into the **Customers** page to provide on-demand AI analysis. 
* **Functionality**: 
  * Takes the current list of customers and their order history as a prompt.
  * Summarizes the top-spending customers.
  * Performs customer segmentation based on purchasing behavior and product categories (e.g., Electronics, Furniture, Books).
  * Returns a structured markdown response which is rendered directly in the UI.

## đźŚŠ Project Flow & Architecture

### 1. Entry Point & Routing
* The application initializes in `src/main.tsx`, wrapping the app in a `BrowserRouter`.
* `src/App.tsx` defines the application routes (`/`, `/customers`, `/orders`, `/communications`) and wraps them in the `Layout` component.

### 2. Layout & Navigation (`src/components/Layout.tsx`)
* Provides a persistent sidebar navigation and a top header.
* Handles responsive design, allowing the sidebar to collapse on smaller screens.

### 3. Core Pages / Views
* **Dashboard (`/`)**: Displays high-level metrics, sales charts, and recent activity.
* **Customers (`/customers`)**: 
  * Displays a list of customers in a card layout.
  * **Nested Orders**: Each customer card expands to show their specific order history, including product categories and totals.
  * **AI Analysis**: Features a "Sparkles" button that triggers the Gemini API to analyze the displayed customer and order data, rendering the insights inline.
* **Orders (`/orders`)**: A searchable, filterable data table displaying all system orders, their status, and totals.
* **Communications (`/communications`)**: A log of emails and messages sent to customers, tracking delivery status and timestamps.

### 4. Data Management
* Currently, the application runs on a mock data layer located in `src/data/mockData.ts`.
* This file exports `customers`, `orders`, and `communications` arrays, simulating a relational database structure where orders are linked to customers via `customerId`.
